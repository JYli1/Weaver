## 报告生成指南

根据本次 session 的交互历史，生成结构化渗透测试报告。

### 报告模板

```markdown
# 渗透测试报告

- 目标: {从 session 中提取}
- 时间: {session 开始时间} ~ {当前时间}
- 操作者: {用户}

## 发现摘要

- 高危: X 个
- 中危: X 个
- 低危: X 个

## 详细发现

### [严重程度] 漏洞名称

- 位置: {URL/IP/端口}
- 描述: {漏洞描述}
- 复现步骤:
  1. ...
  2. ...
- 影响: {潜在影响}
- 建议修复: {修复建议}

## 执行的操作时间线

| 时间 | 操作 | 结果 |
|------|------|------|
| ... | ... | ... |

## 未完成事项

- {待进一步测试的点}
```

### 生成规则

1. 从 session 历史中提取所有执行的命令和结果
2. 识别发现的漏洞并按严重程度分类
3. 按时间顺序整理操作时间线
4. 标注未完成的测试点供后续跟进
5. 保存到配置的报告目录
